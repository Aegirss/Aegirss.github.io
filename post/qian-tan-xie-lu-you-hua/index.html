浅谈斜率优化

2020-02-09 14 min read # DP（斜率优化）

前言：

对于一类形如：Fi=min⁡Li≤j≤Ri{Fj+val(i,j)}F_i=\min_{L_i≤j≤R_i} \{F_j+val(i, j) \}Fi​=minLi​≤j≤Ri​​{Fj​+val(i,j)} 的动态规划模型，我们有两种优化方法：

  1. 当val(i,j)val(i,j)val(i,j)的每一项仅与i,ji,ji,j中的一个有关时，我们可以使用单调队列进行优化。

  2. 当val(i,j)val(i,j)val(i,j)中包含i,ji,ji,j的乘积项时，我们可以使用斜率优化

我们主要关注第二种情况，即如何维护斜率以实现快速的转移。

例题：

任务安排

设：sumTi=∑j=1iTj, sumFi=∑j=1iFjsumT_i=\sum_{j=1}^{i}T_j,\ sumF_i=\sum_{j=1}^{i}F_jsumTi​=∑j=1i​Tj​, sumFi​=∑j=1i​Fj​，FiF_iFi​表示把前iii个任务分成若干批次处理的最小代价。

不难写出：Fi=min⁡0≤j<i{Fj+sumTi∗(sumFi−sumFj)+S∗(sumFN−sumFj)}F_i=\min_{0≤j<i} \{F_j+sumT_i*(sumF_i-sumF_j)+S*(sumF_N-sumF_j) \}Fi​=min0≤j<i​{Fj​+sumTi​∗(sumFi​−sumFj​)+S∗(sumFN​−sumFj​)}

如果只是要通过本题，那么O(n2)O(n^2)O(n2)的算法足矣。但是我们还可以继续优化，最终到达O(n)O(n)O(n)的复杂度。

不妨设两个决策0≤j<k<i0≤j<k<i0≤j<k<i且kkk优于jjj（稍后会解释为什么要设j<kj<kj<k）。

有：Fk+sumTi∗(sumFi−sumFk)+S∗(sumFN−sumFk)<=Fj+sumTi∗(sumFi−sumFj)+S∗(sumFN−sumFj)F_k+sumT_i*(sumF_i-sumF_k)+S*(sumF_N-sumF_k) <= F_j+sumT_i*(sumF_i-sumF_j)+S*(sumF_N-sumF_j)Fk​+sumTi​∗(sumFi​−sumFk​)+S∗(sumFN​−sumFk​)<=Fj​+sumTi​∗(sumFi​−sumFj​)+S∗(sumFN​−sumFj​)

化简移项后有：Fk−Fj<=(sumTi−S)(sumFk−sumFj)F_k-F_j<=(sumT_i-S)(sumF_k-sumF_j)Fk​−Fj​<=(sumTi​−S)(sumFk​−sumFj​)

注意到sumFsumFsumF是单调递增的，我们之前设j<kj<kj<k，所以这里sumFk−sumFj>0sumF_k-sumF_j>0sumFk​−sumFj​>0（除过去不变号）。我们把它除过去，化成一个斜率式。

即：Fk−FjsumFk−sumFj<=SumTi−S\frac{F_k-F_j}{sumF_k-sumF_j}<=SumT_i-SsumFk​−sumFj​Fk​−Fj​​<=SumTi​−S。

注意到这个式子形如：y2−y1x2−x1\frac{y_2-y_1}{x_2-x_1}x2​−x1​y2​−y1​​这样的斜率式。于是把每个决策jjj看成一个坐标为(sumFj,Fj)(sumF_j,F_j)(sumFj​,Fj​)的点。

那么上面式子的意义就是：若j<kj<kj<k且j,kj,kj,k的斜率小于等于sumTi−SsumT_i-SsumTi​−S，则kkk比jjj优。

那要怎么优化呢？我们考虑维护一个决策队列qqq，注意到如果队列里的决策斜率是单调递增的，那么只要队首ql,ql+1q_l,q_{l+1}ql​,ql+1​的斜率大于sumTi−SsumT_i-SsumTi​−S，那么队首就是最优决策。于是我们维护一个斜率单调递增的队列（斜率递增，即一个下凸壳）。

对于队首，我们只需检查ql,ql+1q_l,q_{l+1}ql​,ql+1​的斜率是否小于等于sumTi−SsumT_i-SsumTi​−S，如果小于等于，说明ql+1q_{l+1}ql+1​优于qlq_lql​，则将qlq_lql​出队（由于sumT−SsumT-SsumT−S递增，所以qlq_lql​不会再对之后的sumTi−SsumT_i-SsumTi​−S产生贡献）。

对于队尾，每次转移完iii后，都要往qqq里加入决策点(sumFi,Fi)(sumF_i,F_i)(sumFi​,Fi​)。注意到sumFsumFsumF也是递增的，所以我们只需比较(qr−1,qr,i)(q_{r-1},q_r,i)(qr−1​,qr​,i)这三个点是否满足斜率递增，如果不满足，说明qr−1,qrq_{r-1},q_rqr−1​,qr​的斜率过大，我们将qrq_rqr​出队，继续判断新的队尾是否构成斜率递增关系即可。

每个决策都只会进队和出队一次，所以复杂度是O(n)O(n)O(n)的。

#include<cstdio>
using namespace std;

const double esp = 1e-8;
const int MAXN = 1000010;

int N, S, F[MAXN];
int sumT[MAXN], sumC[MAXN];
int l, r, q[MAXN];

int X(int i) { return F[i]; }

int Y(int i) { return sumC[i]; }

double Slope(int i, int k) { return (double) (X(i) - X(k)) / (Y(i) - Y(k)); }

int main()
{
    scanf("%d %d", &N, &S);
    for(int i = 1; i <= N; ++i)
    {
        scanf("%d %d", &sumT[i], &sumC[i]);
        sumT[i] += sumT[i - 1];
        sumC[i] += sumC[i - 1];
    }
    q[l = r = 1] = 0;
    for(int i = 1; i <= N; ++i)
    {
        while(l < r && Slope(q[l + 1], q[l]) - (S + sumT[i]) < esp) ++l;
        int j = q[l];
        F[i] = F[j] + sumT[i] * (sumC[i] - sumC[j]) + S * (sumC[N] - sumC[j]);
        while(l < r && Slope(i, q[r]) - Slope(q[r], q[r - 1]) < esp) --r;
        q[++r] = i;
    }
    printf("%d\n", F[N]);
    return 0;
}

拓展：

注意到上面的题目有两个特殊性质，即sumT−S,sumFsumT-S, sumFsumT−S,sumF单调递增。第一个性质保证了我们只需保留队列中斜率大于sumT−SsumT-SsumT−S的部分（就是说从队首出队后，对之后的sumT−SsumT-SsumT−S都没有贡献。即出队后不可能再入队），第二个性质告诉我们每次新加入的点一定在最末尾，只需和队尾比较斜率即可。

如果sumT−SsumT-SsumT−S不单调，那我们就不能从队首出队任何元素。由于这是一个单调队列，可以进行二分，在O(logn)O(logn)O(logn)的时间内找到斜率大于当前的sumT−SsumT-SsumT−S的决策点位置。

如果sumFsumFsumF不单调，我们就需要在任何位置插入决策点，可以用平衡树来维护（即维护一个动态凸壳）。

练习：

APIO2010 特别行动队

令sis_isi​为前缀和，fif_ifi​表示前iii个士兵的最大和，显然有：fi=max⁡0≤j<i{fj+calc(i,j)}f_i=\max_{0≤j<i} \{f_j+calc(i,j)\}fi​=max0≤j<i​{fj​+calc(i,j)}

calc(i,j)calc(i,j)calc(i,j)表示j+1 ij+1~ij+1 i的士兵分在一起的代价。

按照套路设0≤j<k<i0≤j<k<i0≤j<k<i，kkk优于jjj。

于是有：(fk+ask2−bsk)−(fj+asj2−bsj)sk−sj≥2asi\frac{(f_k+as_k^2-bs_k)-(f_j+as_j^2-bs_j)}{s_k-s_j}≥2as_isk​−sj​(fk​+ask2​−bsk​)−(fj​+asj2​−bsj​)​≥2asi​

维护一个单调递减的队列。发现2as2as2as是递减的，可以在队首直接进行操纵。因为sss是递增的，所以一定是在队尾插入决策点。

因为满足上面两个性质，直接O(n)O(n)O(n)维护即可。

#include<bits/stdc++.h>
using namespace std;

typedef long long ll;
const int MAXN = 1000010;

int n, a, b, c, l, r;
int x[MAXN], q[MAXN];
ll sum[MAXN], f[MAXN];

double slope(int k, int j)
{
    return (double)((f[k] + a * sum[k] * sum[k] - b * sum[k]) - (f[j] + a * sum[j] * sum[j] - b * sum[j])) / (sum[k] - sum[j]);
}

int main()
{
    scanf("%d %d %d %d", &n, &a, &b, &c);
    for(int i = 1; i <= n; ++i)
    {
        scanf("%d", &x[i]);
        sum[i] = sum[i - 1] + x[i];
    }
    f[0] = q[l = r = 1] = 0;
    for(int i = 1; i <= n; ++i)
    {
        while(l < r && slope(q[l], q[l + 1]) >= 2.0 * a * sum[i]) ++l;
        f[i] = f[q[l]] + a * (sum[i] - sum[q[l]]) * (sum[i] - sum[q[l]]) + b * (sum[i] - sum[q[l]]) + c;
        while(l < r && slope(q[r - 1], q[r]) <= slope(q[r], i)) --r;
        q[++r] = i;
    }
    printf("%lld\n", f[n]);
    return 0;
}

NOI2007 货币兑换

可以证明：一定存在一种最优方案，每次兑换金券花光所有的钱，每次换钱时花掉所有的金券。

因为如果第iii天换成金卷，第jjj天换成钱能够盈利。一定是在第iii天把钱全部换成金卷，第jjj天再全部换成钱最优。

于是设：fif_ifi​表示第iii天的最大收益，Ai,BiA_i, B_iAi​,Bi​分别表示第iii天最多能得到多少金卷A，B。

那么有：Ai=fiRiaiRi+bi, Bi=fiaiRi+bi, fi=max⁡1≤j<i{Aj∗ai+Bj∗bi,fi−1}A_i=\frac{f_iR_i}{a_iR_i+b_i},\ B_i=\frac{f_i}{a_iR_i+b_i},\ f_i=\max_{1≤j<i} \{A_j*a_i+B_j*b_i, f_{i-1}\}Ai​=ai​Ri​+bi​fi​Ri​​, Bi​=ai​Ri​+bi​fi​​, fi​=max1≤j<i​{Aj​∗ai​+Bj​∗bi​,fi−1​}

注意到这里没有单调性，所以我们从另一种角度来推导式子。把转移方程中的maxmaxmax和fi−1f_{i-1}fi−1​忽略，把关于jjj的项作为横纵坐标。

于是得到：Bj=−aibiAj+fibiB_j=-\frac{a_i}{b_i}A_j + \frac{f_i}{b_i}Bj​=−bi​ai​​Aj​+bi​fi​​。可以把式子看作y=kx+by=kx+by=kx+b的形式。

我们相当于是用一条斜率为−aibi-\frac{a_i}{b_i}−bi​ai​​的直线去截每个点，显然当截距最大时，fif_ifi​最大。
数形结合理解一下，我们从上到下平移这条直线，最优决策点其实构成了一个上凸壳（斜率递减）。
对于当前的斜率−aibi-\frac{a_i}{b_i}−bi​ai​​，如果某个点左边的斜率大于它，右边的斜率小于它，这个点就是最优决策。

注意到横坐标，斜率都没有单调性，所以我们还要支持在任意位置插入一个点，即动态维护，查询上凸壳。
我们使用splaysplaysplay来维护这个凸壳，splaysplaysplay里面按照横坐标排序，快速找到当前点的位置，然后往左右比较，维护斜率的单调递减即可。

#include<bits/stdc++.h>
using namespace std;

const int MAXN = 100010;
const double EPS = 1e-9;
const double inf = 1e18;

int n, rt, cnt, fa[MAXN], son[MAXN][2];
double A[MAXN], B[MAXN], lk[MAXN], rk[MAXN], dp[MAXN];
//lk, rk分别表示splay中点k左边，右边直线的斜率

double calc(int a, int b) {if(fabs(A[a] - A[b]) < EPS) return -inf; return (B[a] - B[b]) / (A[a] - A[b]);}
int chk(int x) {return son[fa[x]][1] == x;}

void rotate(int x, int &goal)
{
    int f = fa[x], g = fa[f], k = chk(x);
    if(f == goal) goal = x;
    else son[g][chk(f)] = x;
    fa[f] = x, fa[x] = g, fa[son[x][k ^ 1]] = f;
    son[f][k] = son[x][k ^ 1], son[x][k ^ 1] = f;
}

void splay(int x, int &goal)
{
    while(x != goal)
    {
        if(fa[x] != goal)
        {
            if(chk(x) ^ chk(fa[x])) rotate(x, goal);
            else rotate(fa[x], goal);
        }
        rotate(x, goal);
    }
}

int Find(int x, double res)
{
    if(!x) return 0;
    if(lk[x] + EPS >= res && rk[x] - EPS <= res) return x;
    else if(lk[x] - EPS < res) return Find(son[x][0], res);
    else return Find(son[x][1], res);
}

void ins(int &x, int f, int now)
{
    if(!x) {x = now, fa[x] = f; return;}
    if(A[now] - EPS <= A[x]) ins(son[x][0], x, now);
    else ins(son[x][1], x, now); 
}

int pre(int x)
{
    int y = son[x][0], now = y;
    while(y)
    {
        if(lk[y] + EPS >= calc(y, x)) now = y, y = son[y][1];
        else y = son[y][0];
    }
    return now;
}

int nxt(int x)
{
    int y = son[x][1], now = y;
    while(y)
    {
        if(rk[y] <= calc(y, x) + EPS) now = y, y = son[y][0];
        else y = son[y][1];
    }
    return now;
}

void update(int x)
{
    splay(x, rt);
    if(son[x][0])
    {
        int l = pre(x);
        splay(l, son[x][0]);
        son[l][1] = 0;
        lk[x] = rk[l] = calc(x, l);
    }
    else lk[x] = inf;
    if(son[x][1])
    {
        int r = nxt(x);
        splay(r, son[x][1]);
        son[r][0] = 0;
        rk[x] = lk[r] = calc(x, r);
    }
    else rk[x] = -inf;
    if(lk[x] <= rk[x] + EPS)
    {
        rt = son[x][0], son[rt][1] = son[x][1];
        fa[son[x][1]] = rt, fa[rt] = 0;
        lk[rt] = rk[son[rt][1]] = calc(rt, son[rt][1]);
    }
}

int main()
{
    scanf("%d %lf", &n, &dp[0]);
    for(int i = 1; i <= n; ++i)
    {
        double a, b, r;
        scanf("%lf %lf %lf", &a, &b, &r);
        int j = Find(rt, -a / b);
        dp[i] = max(dp[i - 1], A[j] * a + B[j] * b);
        B[i] = dp[i] / (a * r + b), A[i] = B[i] * r;
        ins(rt, 0, i); update(i);
    }
    printf("%.3f\n", dp[n]);
    return 0;
}
