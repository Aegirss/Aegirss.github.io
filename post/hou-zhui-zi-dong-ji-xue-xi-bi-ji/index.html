定义：

SAM：

后缀自动机可以看成是一张DAG，并且在这张DAG上能表示出一个字符串的所有子串。

当然字典树也可以完成这个任务，但与字典树不同的是，SAM的边数和点数都是O(n)O(n)O(n)的。

endposendposendpos:

这是SAM的精髓，务必要理解清楚。

对于一个字符串，它的一个子串ppp在原串中出现的所有位置的右端点标号组成的集合，我们称之为endpos(p)endpos(p)endpos(p)。比如说原串为aababaababaabab，则endpos(ab)=3,5endpos(ab)={3,5}endpos(ab)=3,5。

神奇的地方在于它的性质：

  1. 如果两个子串的endposendposendpos相同，则其中子串一个必然为另一个的后缀。

  2. 对于任意两个子串t,p (lent≤lenq)t,p\ (len_t≤len_q)t,p (lent​≤lenq​)，要么endpos(q)∈endpos(t)endpos(q) ∈ endpos(t)endpos(q)∈endpos(t)，要么endpos(q)∩endpos(t)=∅endpos(q) ∩ endpos(t) = \emptyendpos(q)∩endpos(t)=∅

  3. 对于endposendposendpos相同的子串，我们将它们归为一个endposendposendpos等价类，则这个等价类里的串的长度连续。

上面三个定理可以从直观上简单理解，严格的证明也很容易给出。下面来看最重要的两个结论：

  1. endposendposendpos等价类的个数为O(n)O(n)O(n)

对于一个endposendposendpos等价类，我们考虑它所包含的最长的子串ppp。在ppp的前面任意添加一个字符所得到的字符串必然不属于此类（否则ppp就不是最长的），从而我们能得到若干个新的类。由前面的结论2可知，新得到的等价类必定是endpos(p)endpos(p)endpos(p)的子集。那么这相当于是对一个集合进行分割（新的集合也可以继续分割），容易发现最终形成的集合个数是O(n)O(n)O(n)的。

我们考虑分割关系，将一个集合所分割出来的新集合作为它的子节点，依次类推我们就能得到一颗树。

我们称这颗树为parent treeparent \ treeparent tree，如下图所示（原串为aababaaababaaababa）：

  1. 对于一个等价类endpos(p)endpos(p)endpos(p)，我们称它最长子串的长度为len(p)len(p)len(p)，最短子串的长度为minlen(p)minlen(p)minlen(p)，fa(p)fa(p)fa(p)表示它的父亲（在parent treeparent \ treeparent tree中）。则：len(fa(p))+1=minlen(p)len(fa(p)) + 1 = minlen(p)len(fa(p))+1=minlen(p)。

根据结论4的证明过程，在一个类前面加一个字符，形成的新类即为它的子节点，由此可知结论5成立。

我们要求的SAM的节点即为parent treeparent \ treeparent tree中的节点，在此基础上，我们还需要添加一些边，最终的成果如下图所示（原串为aababaaababaaababa）：

具体解释过于复杂，可以去看前言中提到的博客中的解释，这里只简单的提一些能帮助理解原文的要点。

  • 沿着fafafa向上走是删除字符串前面的字符，沿着chchch向下走是在字符串后面添加字符。

  • case3的情况实质上就是一个等价类被拆成了两个等价类（不拆就不符合定义了）。

  • 沿着fafafa往上跳的过程本质上是从长到短地，压缩地遍历一个串的所有后缀。

TJOI2015 弦论

设sumisum_isumi​表示经过节点iii的子串个数，sizisiz_isizi​表示节点iii所代表的endposendposendpos集合大小。

两个数组的求法见代码，求出来之后利用它们拼凑出要求的子串即可。

#include<bits/stdc++.h>
using namespace std;

typedef long long ll;
const int MAXN = 1000010;

struct Node
{
	int ch[26];
	int fa, len;
}t[MAXN];
int las = 1, cnt = 1;

ll siz[MAXN];

void build(int c)
{
	int p = las, np = ++cnt; las = cnt;
	t[np].len = t[p].len + 1; siz[cnt] = 1;
	for(; p && !t[p].ch[c]; p = t[p].fa) t[p].ch[c] = np;
	if(!p) t[np].fa = 1;
	else
	{
		int q = t[p].ch[c];
		if(t[q].len == t[p].len + 1) t[np].fa = q;
		else
		{
			int nq = ++cnt; t[nq] = t[q];
			t[nq].len = t[p].len + 1;
			t[q].fa = t[np].fa = nq;
			for(; p && t[p].ch[c] == q; p = t[p].fa) t[p].ch[c] = nq;
		}
	}
}

int n, T, a[MAXN], tmp[MAXN];
char s[MAXN]; ll k, sum[MAXN];

void Print(int x, int K)
{
	if(K <= siz[x]) return;
	K -= siz[x];
	for(int i = 0; i < 26; ++i)
	{
		int y = t[x].ch[i];
		if(!y) continue;
		if(K > sum[y]) {K -= sum[y]; continue;}
		putchar(i + 'a'); Print(y, K); return;
	}
}

int main()
{
	scanf("%s", s + 1); n = strlen(s + 1);
	scanf("%d %lld", &T, &k);
	for(int i = 1; i <= n; ++i) build(s[i] - 'a');
	for(int i = 1; i <= cnt; ++i) ++tmp[t[i].len];
	for(int i = 1; i <= cnt; ++i) tmp[i] += tmp[i - 1];
	for(int i = 1; i <= cnt; ++i) a[tmp[t[i].len]--] = i;
	for(int i = cnt; i; --i) siz[t[a[i]].fa] += siz[a[i]];
	for(int i = 1; i <= cnt; ++i) T == 0 ? (sum[i] = siz[i] = 1) : (sum[i] = siz[i]);
	siz[1] = sum[1] = 0;
	for(int i = cnt; i; --i)
		for(int j = 0; j < 26; ++j)
			if(t[a[i]].ch[j]) sum[a[i]] += sum[t[a[i]].ch[j]];;
	if(sum[1] < k) puts("-1");
	else Print(1, k), puts("");
	return 0;
}

HEOI2015最短不公共子串

先考虑子串，我们可以直接建出SAM。

子序列怎么办？其实我们有后缀自动机，也有序列自动机，而且序列自动机比SAM简单得多。

于是建出两个自动机，同时在它们上面跑广搜，遇到不匹配的即为答案。

下面给出序列自动机的构建方法：

struct SSAM
{
	int nxt[MAXN][26];
	
	void build(char *s)
	{
		int n = strlen(s + 1);
		for(int i = n; i; --i)
		{
			for(int j = 25; ~j; --j)
				nxt[i - 1][j] = nxt[i][j];
			nxt[i - 1][s[i] - 'a'] = i;
		}
	}
}SSAM1, SSAM2;

SDOI2016 生成魔咒

注意到SAM是一个个字符加入的，和题目中的要求刚好吻合，于是考虑使用SAM。

首先新加入了一个字符后，这个字符本身显然是一个新的子串，然后我们只需考虑它的后缀。

考虑在SAM构建函数中往上跳fafafa遍历后缀的情形，当我们找到当前节点npnpnp的fafafa时，那些长度比len[np]−len[fa]len[np] - len[fa]len[np]−len[fa]短的后缀显然都不会产生新的子串。而那些长度大于len[np]−len[fa]len[np] - len[fa]len[np]−len[fa]的后缀加上新字符后都能产生一个新的子串，于是每次添加对答案的贡献即为：len[np]−len[fa]len[np] - len[fa]len[np]−len[fa]。

注意到这个式子很像是求树上两点距离，于是想到parent treeparent\ treeparent tree。

但是parent treeparent\ treeparent tree上两点的lcalcalca是它们的最长公共后缀。开一下脑洞，把字符串反过来建SAM，这样lcalcalca就成了最长公共前缀。

我们设边的权值为len(p)−len(fa(p))len(p) - len(fa(p))len(p)−len(fa(p))，这样就把问题转化成了树上所有路径的边权之和，直接枚举每条边，考虑它的贡献即可。

后缀自动机的用途广泛而且容易构建，能解决许多关于子串，后缀的计数类问题，同时利用parent treeparent\ treeparent tree的性质我们还可以快速的求出最长公共前缀（后缀）。
